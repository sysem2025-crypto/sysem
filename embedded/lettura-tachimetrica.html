Lettura Tachimetrica

Rilevare la velocità del motore con un sensore Hall e un pin di interrupt on-change

Problema

Per regolare la velocità di una ventola AC con triac serve un feedback. Un sensore Hall sul rotore genera un impulso ogni giro (o ogni frazione di giro). Il PIC deve misurare il periodo tra impulsi successivi con risoluzione adeguata, mediare per eliminare fluttuazioni meccaniche, e fornire il valore alla logica di controllo PID. Inoltre deve rilevare condizioni di rotore bloccato o tachimetro guasto per attivare le protezioni.

Soluzione

L’impulso tachimetrico arriva su un pin di PORTB configurato per interrupt on-change. Il gestore cattura il timestamp corrente da Timer1 (16 bit) e lo accumula in un buffer circolare a 4 campioni. Una task periodica (ogni 250 ms) calcola la velocità media e la mette a disposizione del controllore PID. Se non arrivano impulsi per un tempo massimo (FS_ROT_LOCK), scatta la protezione rotore bloccato.

Interrupt handler per il fronte tachimetrico

; Decode_Tacho: chiamato dall'interrupt on-change
; quando il pin TACHO cambia stato (fronte sensore Hall)

Decode_Tacho
    ; Salva il timestamp corrente (Timer1 16 bit)
    movfw   TMR1L
    movwf   Tacho_Timestamp_Lo
    movfw   TMR1H
    movwf   Tacho_Timestamp_Hi

    ; Segnala alla task che un nuovo impulso è arrivato
    SETB    F_TACHO_OK

    goto    End_Test_On_Change

; Nel loop principale dello scheduler, il flag F_TACHO_OK
; viene usato dalle task per sapere se aggiornare la velocità.

Buffer circolare e filtro mobile

; V_Tacho è un buffer di 4 byte in RAM generale.
; Ogni nuovo periodo tachimetrico viene inserito,
; shiftando i precedenti. La somma dei 4 campioni
; rappresenta il periodo medio su 4 giri.

; Struttura buffer:
; V_Tacho+0: periodo giro corrente
; V_Tacho+1: periodo giro -1
; V_Tacho+2: periodo giro -2
; V_Tacho+3: periodo giro -3

; Task #Tacho.h (eseguita dallo scheduler ogni 250 ms):
; Calcola la somma dei 4 periodi
    movlw   V_Tacho
    movwf   FSR

    movfw   INDF              ; periodo corrente
    incf    FSR,same
    addwf   INDF,w            ; + periodo -1
    incf    FSR,same
    addwf   INDF,w            ; + periodo -2
    incf    FSR,same
    addwf   INDF,w            ; + periodo -3 = somma 4 periodi

    movwf   RB0_0             ; velocità normalizzata

; Il valore è inversamente proporzionale alla velocità
; (più alto = più lento). La conversione in RPM
; avviene nel modulo PID (vedi Controllo Triac).

Protezione rotore bloccato

; Nella task Triac_Power (eseguita ogni 1 sec):
; Se la somma tachimetrica è troppo alta (rotore lento o fermo),
; si incrementa un timer dedicato T_Rot_Lock.
; Se T_Rot_Lock raggiunge FS_ROT_LOCK, scatta allarme.

Test_Rotore
    movlw   VEL_MIN_ROT_LOCK
    subwf   RB0_0,w           ; confronto velocità con soglia
    skpc
    goto    Test_Rot_Lock     ; troppo lento

    ; Velocità OK: reset timer protezione
    BANK    1
    clrf    T_Rot_Lock
    BANK    0
    goto    Test_Vel_Ventola

Test_Rot_Lock
    BANK    1
    movlw   FS_ROT_LOCK
    xorwf   T_Rot_Lock,w
    BANK    0
    skpnz
    SETB    ALARM_NTC         ; allarme rotore bloccato!
    ; L'allarme ferma la macchina e accende led di errore

Schema del sistema di misura

|  Motore  |----+---------->| PORTB (RB7)       |

|  + Hall  |    |             | Change            |

                |             +-------------------+
                |                      |
                |                      v

                |             |   salva Timer1    |
                |             |   set F_TACHO_OK  |

                |                      |
                |          ogni 250ms  v
                |             +-------------------+
                |             | Task Tacho        |
                |             |   media 4 campioni |

                |             |   protezione      |

                v                      v
           Pw_State2            ALARM_NTC
           (ritardo triac)      (led errore)

Lezioni apprese

  • Media mobile a 4 campioni: smorza le fluttuazioni meccaniche della ventola senza ritardare eccessivamente la risposta del PID
  • Timer1 a 16 bit: risoluzione di ~1 µs sul periodo, sufficiente per discriminare velocità da 200 a 1500 RPM
  • Protezione integrata: il rotore bloccato viene rilevato in ~4 secondi e ferma la macchina (non si surriscalda il motore)
  • Ereditarietà tra progetti: lo stesso identico approccio è usato nell’Antisbilanciamento (I200026) per rilevare lo sbilanciamento in centrifugaLettura Tachimetrica
