    Controllo Triac e Utenze

    Azionamento di carichi AC in ambiente domestico: taglio di fase, sincronismo di rete, zippillo a tempo da interrupt

    Problema

    In uno split system, le ventole interna ed esterna sono motori AC monofase la cui velocità viene regolata tramite taglio di fase (phase-cut). Un triac, innescato a un ritardo variabile dopo lo zero crossing della rete, conduce solo per una frazione della semi-onda. Più piccolo è il ritardo, più potenza arriva al carico. Servono: sincronismo con la rete (50 Hz), generazione dell’impulso di gate (zippillo) con durata precisa, e aggiornamento continuo del ritardo in base alla velocità desiderata.

    Soluzione

    Due meccanismi distinti e complementari:

    • Interrupt on-change su SYNC: il fronte di zero crossing della rete (rilevato da un optoaccoppiatore) genera un interrupt che resetta i timer di innesco
    • Interrupt Timer0: viene usato come base dei tempi per lo scheduler OS e per generare lo zippillo del triac con temporizzazione di 21 µs

    Rivelazione sincronismo di rete

    ; Interrupt on-change su PORTB rileva il fronte di SYNC
; (pin RB5 collegato a optoaccoppiatore rete)

Decode_Sync
    ; Salva il ritardo triac corrente come punto di partenza
    ; per il prossimo semiperiodo
    movfw   Pw_State2
    movwf   Sync_Reg          ; ritardo triac per questo semiperiodo

    ; Resetta contatori per innesco
    clrf    TMR0              ; timer0 riparte da 0 per il nuovo semiperiodo
    bcf     INTCON,T0IF

    ; Segnala allo scheduler che il sincronismo è avvenuto
    SETB    F_SYNC_OK
    goto    End_Test_On_Change

    Innesco triac (zippillo) da interrupt Timer0

    ; L'interrupt Timer0 serve sia come passo dello scheduler OS
; sia per generare l'impulso di gate del triac.
; Quando Timer0 raggiunge il valore programmato (ritardo),
; il triac viene acceso e mantenuto per 21 us (zippillo).

Test_Timer0
    btfss   INTCON,T0IF
    goto    End_Interrupt
    bcf     INTCON,T0IF

    ; Test richiesta ventola interna
Test_V_Int_On
    SSET    V_INT             ; consenso ventola interna acceso?
    goto    Test_V_Ext_On
    SSET    V_INT_REQ         ; richiesta attivazione?
    goto    Test_V_Ext_On
    SETB    OK_V_INT          ; impulso di gate consentito
    CLRB    IO_V_INT          ; pin basso = triac ON (innesco)

Test_V_Ext_On
    SSET    V_EXT             ; consenso ventola esterna?
    goto    Fire_On_Triac
    SSET    V_EXT_REQ
    goto    Fire_On_Triac
    SETB    OK_V_EXT
    CLRB    IO_V_EXT

; Impulso di gate: 21 microsecondi esatti
Fire_On_Triac
    movlw   7
    movwf   Wait_Rit_Triac

Loop_V_xx
    decfsz  Wait_Rit_Triac,same
    goto    Loop_V_xx         ; 3 cicli * 7 = 21 us a 4 MHz

    SETB    IO_V_INT          ; fine impulso: triac OFF
    SETB    IO_V_EXT

    ; ... prossimo passo scheduler OS ...

    goto    End_Interrupt

    Schema temporale del taglio di fase

    Rete 50 Hz (10 ms per semiperiodo):
  +---+     +---+     +---+
  |   |     |   |     |   |
--+   +-----+   +-----+   +-----
  |   |     |   |     |   |
  |   |     |   |     |   |
  ^         ^         ^
  |         |         |
 SYNC      SYNC      SYNC     (fronte di zero crossing)
  |         |         |
  |<-------->|<------->|
  | ritardo  | ritardo  |      (tempo prima dell'innesco)
  |   _      |   _      |
  +--+ +-----+--+ +-----+--   (impulso gate 21 us)
  |  | |     |  | |     |
  +--+ +-----+--+ +-----+--   (conduzione triac - tratteggiato)

  |<------------->|
   periodo di conduzione
   maggiore ritardo = minore conduzione = minore potenza

    Calcolo del ritardo in base alla velocità (PID software)

    ; Il ritardo triac viene calcolato ogni secondo
; con un algoritmo proporzionale-integrale software.

Triac_Power
    ; Legge velocità tachimetrica (4 campioni sommati)
    movlw   V_Tacho
    movwf   FSR
    movfw   INDF
    incf    FSR,same
    addwf   INDF,w
    incf    FSR,same
    addwf   INDF,w
    incf    FSR,same
    addwf   INDF,w
    movwf   RB0_0             ; somma 4 letture tacho

    ; Protezione rotore bloccato
    movlw   VEL_MIN_ROT_LOCK
    subwf   RB0_0,w
    skpc
    goto    Test_Rotore       ; velocità troppo bassa

    ; Limita tra START_RIT e END_RIT
    DEC_BIN_OVF  Integr_Term, END_RIT
    INC_BIN_OVF  Integr_Term, START_RIT

    ; Aggiorna ritardo triac per il prossimo semiperiodo
    movwf   Pw_State2

    ; Conversione: START_RIT = massima potenza (ritardo minimo)
    ;              END_RIT   = minima potenza (ritardo massimo)
    ; START_RIT = 3000 us  -> ventola a fondo
    ; END_RIT   = 7000 us  -> ventola al minimo

    Lezioni apprese

    • Due world separati: il sincronismo rete (SYNC) vive nell’interrupt on-change, il calcolo del ritardo vive nella macchina a stati. Si incontrano solo nel registro Pw_State2
    • Zippillo a 21 us: la durata dell’impulso di gate deve essere sufficiente a innescare il triac ma non deve superare il semiperiodo. 21 us è un valore collaudato
    • Ritardo minimo di sicurezza: il triac non viene mai innescato a meno di 3 ms dallo zero crossing, per evitare di superare il di/dt massimo
    • Controllo integrale: il termine integrale compensa le variazioni di carico meccanico (filtri sporchi, ventole sbilanciate) senza intervento manuale
