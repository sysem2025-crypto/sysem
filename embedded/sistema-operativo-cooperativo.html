Scheduler Semplificato Round-Robin

Scheduler a task multiple con wake-up a tempo o a evento su PIC a 8 bit, senza RTOS esterno

Problema

Un sistema embedded per climatizzazione deve eseguire periodicamente una serie di attività: leggere temperature, controllare tasti, aggiornare display, gestire il triac, comunicare via seriale. Su PIC16Cxx senza RTOS, il codice rischia di diventare un loop monolitico dove ogni attività blocca le altre. Serve un scheduler cooperativo che garantisca a ogni task un timeslice regolare, senza overhead di un RTOS completo.

Soluzione

Entrambi i progetti Split (Ferroli I200036 e Elettroplastica I200052) implementano un sistema operativo cooperativo round-robin in circa 100 righe di Assembly. Ogni task può essere risvegliata:

  • A tempo: dopo un certo numero di passi dello scheduler (timeout decrementante)
  • A evento: quando un flag globale viene attivato (es. interrupt su cambio porta)

Implementazione: OS.H (Split Elettroplastica I200052)

;----------------------------------------------------------------------
; Scheduler cooperativo round-robin
;----------------------------------------------------------------------
#define     MAX_TASK    11            ; numero massimo di task
#define     MSEC_05_OS  16            ; attesa base ~0.5 msec

Operative_System

Loop_OS_Step
    BANK    0
    movfw   Old_Timer0
    subwf   TMR0,w
; Se sono passati almeno MSEC_05_OS tick dal ciclo precedente,
; eseguo un passo dello scheduler
    clrc
    sublw   MSEC_05_OS
    skpnc
    goto    Loop_OS_Step              ; non ancora: aspetto

    movfw   TMR0
    movwf   Old_Timer0                ; aggiorno riferimento

Loop_OS_Task
    BANK    0
    clrwdt                            ; reset watchdog

    incf    Curr_Task,same            ; task successiva
    movlw   MAX_TASK
    xorwf   Curr_Task,W               ; ho superato l'ultima?
    clrc
    skpnz
    clrf    Curr_Task                 ; se sì, riparto da 0

    rlf     Curr_Task,w               ; indice * 2 (tabella 16 bit)
    addlw   Ret_Add + 1               ; puntatore alla tabella ritorni
    movwf   FSR
    movfw   INDF                      ; leggo MSB del return address
    andlw   0x80                      ; MSB=1 significa "wake a evento"
    skpz
    goto    Test_Conditional_Flag     ; task a evento

; Task a tempo: decremento contatore
    movfw   Curr_Task
    addlw   Wake_Up
    movwf   FSR
    tstf    INDF
    skpz
    decfsz  INDF,same                 ; --contatore; se 0, esegui task
    goto    Next_Task
    goto    Esegui_Task               ; scade il timer: esegui!

Test_Conditional_Flag
; Task a evento: controllo flag di risveglio
    movfw   Curr_Task
    addlw   Wake_Up
    movwf   FSR
    movfw   Flag_Task
    andwf   INDF,w
    skpnz
    goto    Next_Task                 ; evento non accaduto: salta

Esegui_Task
    rlf     Curr_Task,w
    addlw   Ret_Add + 1
    movwf   FSR
    movfw   INDF                      ; MSB indirizzo
    movwf   PCLATH
    decf    FSR,same
    movfw   INDF                      ; LSB indirizzo
    movwf   PCL                       ; salto alla task!

Next_Task
    bsf     INTCON,GIE
    BANK    0
    tstf    Curr_Task
    skpnz
    goto    Operative_System          ; giro completato
    goto    Loop_OS_Task              ; prossima task

Struttura dati dello scheduler

; Tabella Wake_Up: per ogni task, il contatore di timeout
; Se MSB di Ret_Add = 0, il task è a tempo (contatore decrementato)
; Se MSB di Ret_Add = 1, il task è a evento (flag testato)
Wake_Up     DS  MAX_TASK     ; contatori indipendenti per ogni task

; Tabella Ret_Add: indirizzi di rientro per ogni task
; Se il MSB ha il bit 7 = 1, lo scheduler usa la modalità evento
Ret_Add     DS  MAX_TASK*2   ; 2 byte per indirizzo (PCLATH + PCL)

; Le task dichiarano la loro periodicità con le macro:
;   WAIT  TASK_NTC, OS_MSEC_100    ; risveglio dopo 100 passi
;   EVENT TASK_NTC, NTC_OK         ; risveglio su evento
;
; La macro WAIT inizializza il contatore nella tabella Wake_Up.
; La macro EVENT programna il flag di risveglio in Flag_Task.

Schema a blocchi

+-------------------+
| RESET             |
| GOTO Startup      |
+-------------------+
         |
         v
+-------------------+
| Startup           |
| Init HW & Var     |
+-------------------+
         |
         v
+-------------------+
| Operative_System  |<-------------------+
|   attesa ~0.5ms   |                    |
|   (poll TMR0)     |                    |
+-------------------+                    |
         |                               |
         v                               |
+-------------------+                    |
| Loop_OS_Task      |                    |
|   Curr_Task++     |                    |
|   se == MAX_TASK  |                    |
|     Curr_Task = 0 |                    |
+-------------------+                    |
         |                               |
         +--> Task a tempo?              |
         |      SI: decrementa cont.     |
         |          se 0 -> esegui       |
         |      NO: Task a evento?       |
         |          SI: flag attivo?     |
         |              se SI -> esegui  |
         |          NO: prox task        |
         |                               |
         +--> Esegui_Task                |
              carica PCLATH, PCL         |
              da Ret_Add[Curr_Task]      |
              -> esecuzione task         |
         |                               |
         v                               |
+-------------------+                    |
| Next_Task         |--------------------+
| se Curr_Task == 0 |
|   goto OS_Step    |
| altrimenti        |
|   goto Loop_Task  |
+-------------------+

Esempio pratico: task NTC

; La task NTC usa WAIT per essere eseguita ogni 100 ms
; e EVENT per sincronizzarsi con l'interrupt di carica condensatore

NtcTask
    WAIT    TASK_NTC, OS_MSEC_100    ; 100 ms di timeout

; ... configurazione pin, avvio carica RC ...

    EVENT   TASK_NTC, NTC_OK         ; aspetta che l'interrupt segnali
                                     ; il completamento della carica
; ... lettura timer, calcolo temperatura ...

Questa combinazione di polling periodico e wake su evento permette a tutte le 11 task di convivere in pochi KB di ROM, con latenze deterministiche e un consumo CPU minimo.

Lezioni apprese

  • 11 task in ~100 righe: lo scheduler è estremamente compatto: circa 95 word di ROM
  • Due modalità di risveglio: tempo (polling) ed evento (interrupt-driven) si combinano per coprire ogni esigenza
  • Zero overhead di contesto: le task collaborano volontariamente (nessun preemption), non serve salvare/ripristinare registri
  • Watchdog integrato: clrwdt a ogni passo dello scheduler protegge da crash
  • Frequenza regolabile: la costante MSEC_05_OS definisce il passo base (tipicamente 0.5-2 ms)
